# voice-hermes-plugin

[中文](./README.md) | [English](./README.en.md)

Xalgo Voice Hermes platform plugin. It connects Xalgo glasses to Hermes Agent
through the same Xalgo voice channel protocol used by `voice-openclaw-plugin`.

## Features

This plugin connects Xalgo glasses voice input to Hermes Agent:

- Uses the Hermes platform plugin layout: `plugin.yaml` + `__init__.py` + `adapter.py`
- Connects outbound to Xalgo Voice Channel Server, so Hermes does not need a public port
- Exchanges an 8-character Xalgo binding code for a long-lived channel token
- Supports WebSocket `connect` / `resume` / `ping` / `pong`
- Converts Xalgo voice turns into Hermes messages and Hermes replies back into Xalgo voice output
- Supports streaming replies, delivery acks, voice interrupts, token rotation, and binding revoke events

## Installation

### 1. Install Python dependencies

Skip this step if your Hermes environment already includes `httpx` and `websockets`.

```bash
pip install -r /Users/leo/project/voice-hermes-plugin/requirements.txt
```

Or install them directly:

```bash
pip install httpx websockets
```

### 2. Install the plugin into Hermes

Symlink is recommended during development:

```bash
mkdir -p ~/.hermes/plugins
ln -s /Users/leo/project/voice-hermes-plugin ~/.hermes/plugins/xalgo-voice-platform
```

If the target host cannot access this project path, copy the directory instead:

```bash
mkdir -p ~/.hermes/plugins/xalgo-voice-platform
cp -R /Users/leo/project/voice-hermes-plugin/* ~/.hermes/plugins/xalgo-voice-platform/
```

### 3. Enable the plugin

Edit `~/.hermes/config.yaml` and add the plugin to `plugins.enabled`:

```yaml
plugins:
  enabled:
    - xalgo-voice-platform
```

If you already have enabled plugins, append `xalgo-voice-platform` to the existing list.

### 4. Bind your Xalgo account

Run Hermes gateway setup and choose `Xalgo Voice`:

```bash
hermes gateway setup
```

The setup flow asks for:

- Xalgo REST API base URL, defaulting to this repo's `endpoints.json`
- The 8-character binding code generated by the Xalgo App
- Device label, for example `Hermes on Mac`

After a successful bind, the plugin writes these values to `~/.hermes/.env`:

```bash
XALGO_VOICE_TOKEN=...
XALGO_VOICE_INSTANCE_ID=...
XALGO_VOICE_SERVER_URL=wss://...
XALGO_VOICE_API_BASE_URL=https://...
XALGO_VOICE_BOUND_USER_ID=...
XALGO_VOICE_BOUND_USER_NAME=...
XALGO_VOICE_DEVICE_LABEL=...
```

### 5. Restart and verify

Restart the Hermes gateway:

```bash
hermes gateway restart
```

Check status:

```bash
hermes gateway status
```

Gateway logs should show that Xalgo Voice was loaded and the WebSocket was
connected/authenticated. You can then speak through Xalgo glasses to trigger
Hermes Agent.

## Manual Configuration

You can skip setup and write the required env vars into `~/.hermes/.env`:

```bash
XALGO_VOICE_TOKEN=<channel-token>
XALGO_VOICE_INSTANCE_ID=<stable-instance-id>
XALGO_VOICE_SERVER_URL=wss://asr-test.jlpay.com/openclaw/connect
XALGO_VOICE_API_BASE_URL=https://asr-test.jlpay.com
XALGO_VOICE_DEVICE_LABEL="Hermes on Mac"
```

Optional settings:

```bash
XALGO_VOICE_REPLY_MODE=voice_first  # voice_first | text_first | both
XALGO_VOICE_STREAMING=true
XALGO_VOICE_HOME_CHANNEL=xalgo:user:default
```
